こんにちは。  
ご機嫌いかがでしょうか。  
"No human labor is no human error" が大好きな[吉井 亮](https://twitter.com/YoshiiRyo1)です。  

2020年7月に公開しました非機能要件ヒアリングシートを久しぶりに更新しました。
改めて更新日付を見て、5年以上更新していないという事実を目の当たりにし、反省というか恥ずかしい気持ちというかそういった複雑な感情が入り混じりました。

今回は時代の流れにのって、Claude Code にお手伝いしてもらいました。

1. 陳腐化した項目をピックアップ
2. それらを最新化
3. 2020年7月以降にリリースされたサービスを反映
4. Well-Architected フレームワーク観点でレビュー
5. レビューを反映
6. 満足するまで繰り返しレビューと修正

陳腐化チェックと WA レビューの2つの Skill を作成し、将来の更新を楽にしたつもりです。今後は年1回くらいは更新できるかもしれません。
出力は私のほうで人力レビューはしていますが、もし誤りがあったら修正していただけると助かります。そして、全然アップデートを追っていないなと実感しました。知らないサービスたくさんありました笑。

GitHub で公開しているのでクローンしてお使いください。

https://github.com/YoshiiRyo1/document-templates-for-aws

## 追加・新規作成した質問一覧

### 信頼性 (+11問)

**稼働率（+2問）**
1. システム全体のコンポジット SLA（複合 SLA）を算出していますか？
2. 利用する AWS サービスの Service Quotas（サービスクォータ）を把握していますか？引き上げ申請は実施済みですか？

**バックアップ / リストア（+3問）**
3. クロスアカウントまたはクロスリージョンへのバックアップコピーは必要ですか？
4. イミュータブルバックアップ（変更不可バックアップ）は必要ですか？
5. バックアップデータの論理的エアギャップ保護（Logically Air-Gapped Vault）は必要ですか？

**災害対策（+3問）**
6. どの DR 戦略パターンを採用しますか？（Backup & Restore / Pilot Light / Warm Standby / Multi-Site Active-Active）
7. レジリエンステスト（カオスエンジニアリング）を実施する予定はありますか？
8. DR 訓練の頻度と実施体制を定義していますか？

**アプリケーション回復力（新セクション +3問）**
9. アプリケーションに障害耐性パターン（リトライ、サーキットブレーカー、タイムアウト、グレースフルデグラデーション等）は実装されていますか？
10. API やサービス間通信にスロットリング（流量制御）の仕組みはありますか？
11. システムが依存する外部サービスや AWS サービスの一覧を把握していますか？依存先の障害時の影響を分析していますか？
